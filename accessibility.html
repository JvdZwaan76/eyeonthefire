Accessibility Statement

Our commitment to providing an accessible wildfire tracking platform for all users

Accessibility Commitment

Eye on the Fire is committed to ensuring that our wildfire tracking platform is accessible to everyone, including users with disabilities. We believe that life-saving emergency information should be available to all members of our community.

Our Accessibility Standards

We strive to comply with the Web Content Accessibility Guidelines (WCAG) 2.1 Level AA standards and the Americans with Disabilities Act (ADA) requirements for digital accessibility.

WCAG 2.1 Level AA

We follow international standards for web accessibility, ensuring our platform works with assistive technologies and provides multiple ways to access information.

ADA Compliance

Our platform is designed to meet the accessibility requirements of the Americans with Disabilities Act for digital services.

Section 508

We incorporate Section 508 standards to ensure compatibility with federal accessibility requirements.

Emergency Accessibility

Special consideration is given to accessibility during emergency situations when information access is most critical.

Accessibility Features

Screen Reader Compatibility

  • Semantic HTML markup for proper content structure
  • ARIA labels and landmarks for navigation
  • Alt text for all informative images and icons
  • Live regions for dynamic content updates
  • Descriptive headings and page structure
  • Skip links for efficient navigation

Keyboard Navigation

  • Full keyboard accessibility without mouse dependency
  • Visible focus indicators on all interactive elements
  • Logical tab order throughout the interface
  • Keyboard shortcuts for common actions
  • Escape key functionality for closing dialogs
  • Enter and Space key support for custom buttons

Keyboard Shortcuts

Skip to main content Alt + 1
Find your location Ctrl + L
Refresh fire data Ctrl + R
Open navigation menu Ctrl + M
Show all fires Ctrl + A
Close dialogs/menus Escape

Visual Accessibility

  • High contrast color schemes that meet WCAG standards
  • Minimum 4.5:1 contrast ratio for normal text
  • Minimum 3:1 contrast ratio for large text and graphics
  • Color is not the only means of conveying information
  • Scalable text up to 200% without horizontal scrolling
  • Support for browser zoom up to 500%
  • Distinct focus indicators with sufficient contrast

Motor Accessibility

  • Large touch targets (minimum 44px) for mobile users
  • Adequate spacing between clickable elements
  • Support for alternative input methods
  • Drag and drop interactions have keyboard alternatives
  • No time limits on critical safety information
  • Gesture-based interactions have alternative methods

Cognitive Accessibility

  • Clear, consistent navigation throughout the platform
  • Simple, plain language for emergency instructions
  • Consistent layout and design patterns
  • Error messages with clear instructions for correction
  • Important information is prominently displayed
  • Progress indicators for multi-step processes
  • Help text and tooltips for complex features

Emergency Accessibility Considerations

During wildfire emergencies, accessibility becomes even more critical. We have implemented special features for emergency situations:

Emergency Alerts

  • Screen Reader Announcements: Critical alerts are announced immediately to screen reader users
  • Visual Alerts: High-contrast emergency banners with clear iconography
  • Multiple Notification Methods: Information provided through visual, auditory, and text formats
  • Persistent Alerts: Emergency information remains accessible even during navigation

Evacuation Information

  • Step-by-Step Instructions: Clear, numbered evacuation procedures
  • Priority Information: Most critical information presented first
  • Multiple Formats: Information available in text, audio, and visual formats
  • Offline Access: Critical safety information works without internet connection

Known Accessibility Limitations

While we strive for full accessibility, we acknowledge some current limitations that we are actively working to address:

Interactive Map Features

  • Current Limitation: Some map interactions may be challenging for users with motor disabilities
  • Workaround: Alternative text-based fire information is provided
  • Timeline: Enhanced map accessibility planned for Q2 2025

Real-Time Data Updates

  • Current Limitation: Frequent data updates may be disruptive to some screen readers
  • Workaround: Users can pause live updates in settings
  • Timeline: Improved live region management in development

Complex Data Visualizations

  • Current Limitation: Some data charts may not be fully accessible
  • Workaround: Data tables with the same information are provided
  • Timeline: Enhanced chart accessibility planned for Q3 2025

Third-Party Accessibility

Our platform integrates with several third-party services that may have their own accessibility considerations:

  • Map Services: We provide alternative text-based location information
  • Emergency Contacts: Direct links to accessible emergency services
  • Social Media: Alternative communication methods available
  • Advertising Partners: We work with partners to ensure accessible ad content
